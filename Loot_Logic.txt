Where does loot information exist?

   ┌──────────────────────────┐
   │  AzerothCore MySQL DB    │  ← THE source. You don't edit it for the
   │  (creature_loot_template │     site; you play the game / change loot
   │   item_template)         │     server-side, then rebuild.
   └──────────────────────────┘
              │
              │  build_dungeons.py reads from DB
              │  + uses build_config.yaml for curation
              │  + applies quality_floor filter
              │  + applies smart_filter_block.js rules
              ▼
   ┌──────────────────────────┐
   │   Builder transforms     │
   └──────────────────────────┘
              │
              ▼
   ┌──────────────────────────┐
   │   Static output files    │
   │   - index.html (JSON)    │  ← the Find Loot tab reads this
   │   - 9 × dungeon HTMLs    │  ← per-dungeon pages read this
   └──────────────────────────┘

Mechanically, two output places:

index.html has a JSON blob (<script id="loot-data">) used by the Find Loot tab
Each dungeon HTML has its own pre-rendered loot rows used by that dungeon page
